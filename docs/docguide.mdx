The main purpose of this documentation is to guide users and contributors about
how the Try Token project works. The documentation covers the main topics necessary
to understand the showcase and how to use it in order to get the most out of use
cases.

This documentation is split into 5 sections:

1. **Architecture:** Contains details about the project architecture, how it is
   divided and what does each part.

2. **Developer Guide:** Information about the developer can run the project.

3. **User Guide:** Contains guidelines to explain in detail to users how to
   use the different features of the showcase. 

4. **Adoptions Guide:** This section helps you getting started through the ejection process of the modules included on this project so that you can adopt them on your stack.

5. **Contributor's Guide:** Explains how the developers can help in the project.
